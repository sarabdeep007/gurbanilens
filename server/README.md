# GurbaniLens server

Optional Hetzner-hosted server that runs Whisper-large-v3 for devices
that can't (or don't want to) run on-device ASR, and receives opt-in
correction feedback for future fine-tuning. **Not deployed yet.** This
directory ships the source-available skeleton so the privacy contract
is in writing before any audio leaves a device.

The Phase 2A v1 iOS app does **not** require this server — it ships
with on-device whisper.cpp. The server is the opt-in accuracy upgrade
path for older devices and the feedback channel for users who choose
to help improve recognition.

---

## Privacy contract — enforced in code

These guarantees are baked into the code. Read `src/middleware/privacy.js`
end-to-end; that file *is* the privacy policy. If a line of text below
isn't reflected in code, file an issue — it's a bug, not just a
documentation gap.

### 1. No audio storage

Audio bytes flow through the server as a per-request temp file (path
under `/tmp/gurbanilens-asr/`, randomly-named). The handler deletes the
file in a `finally{}` block so success, error, **and cancellation** all
clean up. Nothing reaches a persistent disk, log line, or queue.

Audio submitted via the feedback channel's `audio_base64` field is
**dropped on the floor** in v1 — the column `audio_blob_path` exists in
the schema but stays NULL. Audio persistence for the future fine-tuning
dataset is a deliberate Phase 3 decision; turning it on requires a code
change and a fresh review.

### 2. No content logging

The Fastify logger uses a tight serializer (`src/middleware/privacy.js`,
`privacyLoggerConfig`). The only fields logged per request are:

```
method | url | statusCode | responseTime
```

No headers (including Authorization), no IP addresses, no request body,
no response body, no transcripts, no cookies. Adding any field to that
projection requires reviewing this section.

### 3. No user identifier

Authentication uses an opaque, ephemeral session token generated by the
app. It is:

- not tied to any user account
- not persisted in the rate-limit bucket beyond the 1-hour window
- never written to disk in raw form

The feedback queue stores `HMAC(session_token, FEEDBACK_HMAC_SECRET)`
instead of the raw token. The HMAC lets a live session find its own
records (`GET /feedback/submissions?session_token=...`); after the
session ends and the client forgets its token, the HMAC value in the
DB is unlinkable to any device or user.

### 4. DE jurisdiction

Deployment target is Hetzner Falkenstein (Germany). GDPR-aligned. No US
Cloud Act exposure. See `DEPLOY.md`.

### 5. Source-available

This directory ships under the same OSS licence as the rest of the
project. Anyone can audit that the running code matches the policy.

### 6. Opt-in per session

The app prompts before any audio leaves the device. Server fallback is
off by default. Feedback channel is off by default.

### 7. Strict request/response only — no telemetry

There is no analytics call, no usage beacon, no error reporter
("Sentry-style") that ships outside this repo. The server speaks HTTP
to one client at a time, replies, and forgets.

### 8. No IP-based rate limit, no IP-based bans

Rate limiting is keyed off the session token, not the IP. We do not
read `req.ip`, `X-Forwarded-For`, `X-Real-IP`, or `Forwarded`. The
reverse proxy (Caddy) strips IP headers before they reach us;
`trustProxy: false` in Fastify is the in-process belt-and-braces.

### 9. No third-party SDKs in the request path

`package.json` runtime dependencies: `fastify`, `@fastify/multipart`,
`better-sqlite3`, `zod`. Each is npm-audited at build time. The Python
worker dependency is `faster-whisper`. That's the entire surface that
sees user audio or transcript text.

## Endpoints

### `POST /transcribe`

One-shot voice-search ASR. Tap-to-speak. **Not** continuous streaming
(that's Phase 2A v2 scope).

- Multipart `audio` field, WAV/MP3/M4A/FLAC/OGG, ≤ 60 s, ≤ 10 MB
- `Authorization: Bearer <opaque-session-token>`
- 60 requests / hour / session
- Returns `{ "transcript": "...", "language": "pa", "duration": 11.0, "model": "large-v3" }`

### `POST /feedback/correction`

Opt-in user correction. Session token in body (per
`docs/feedback_channel_spec.md`).

```jsonc
{
  "session_token": "ephemeral-uuid",
  "app_version": "0.1.0",
  "platform": "ios-17.4",
  "model_size": "small",
  "audio_base64": "...",            // accepted in v1 but DROPPED — not persisted
  "audio_duration_sec": 7.2,
  "audio_codec": "opus",
  "match":      { "ang": 462, "pangti": 3, "score": 64.2, ... },
  "correction": { "type": "wrong_pangti", "ang": 462, "pangti": 5 },
  "mode": "voice_search",
  "matcher_window_text_latin": "..."
}
```

- 20 requests / hour / session
- Returns `202 Accepted` `{ "id": "uuid", "status": "accepted" }`
- Persisted to SQLite at `data/feedback.db`

### `GET /feedback/submissions?session_token=...`

Lists this session's prior submissions (`id`, `received_at_ms`,
`matched_ang/pangti`, `corrected_ang/pangti`, `correction_type`, `mode`).
Empty for any token that hasn't submitted within this DB lifetime.

### `DELETE /feedback/submissions?session_token=...[&id=...]`

Without `id`: deletes every submission for this session. With `id`:
deletes one specific row. Returns `{ "deleted": <count> }`.

### `GET /healthz` / `GET /readyz`

Liveness + readiness. `/readyz` 503s if the Python venv or worker
script is missing.

## Running locally (dev)

```bash
cd server

# 1) Node deps
npm install

# 2) Python venv for faster-whisper
#    (requires `uv` — https://github.com/astral-sh/uv — or `python3 -m venv` if available)
uv venv .venv-asr
uv pip install --python .venv-asr/bin/python faster-whisper

# 3) Config
cp .env.example .env
# Edit .env if you want a non-default port or model.

# 4) Run
npm run dev   # node --watch src/index.js
```

Dev escape hatches:

- `WHISPER_DISABLED=1` — `/transcribe` returns 503 without touching Python
- `WHISPER_MODEL=tiny` — use the 75 MB model instead of the 3 GB large-v3
- `FEEDBACK_DISABLED=1` — `/feedback/*` returns 503

Smoke test (any opaque token will do for auth in dev):

```bash
curl -X POST http://127.0.0.1:4040/transcribe \
  -H "Authorization: Bearer dev-session-abc123" \
  -F "audio=@test-fixtures/jfk.flac"
```
